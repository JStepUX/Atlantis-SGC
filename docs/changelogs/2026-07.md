# 2026-07

## 2026-07-04 ‚Äî `export --archive`: retire the corpus with the build

New flag makes the end-of-loop cleanup programmatic instead of manual
diligence: after a successful pack write, the raw docs, chunk files, and
index.json (the provenance stamp belongs to its build) MOVE into
`<archive_dir>/<pack-id>_<stamp>/` alongside a copy of the exported pack ‚Äî
one self-contained bundle per brain, working dirs left empty for the next
one. Prevents the two silent mistakes the manual flow allowed: stale chunks
bleeding into the next pack, and a stale backend stamp mislabeling the next
corpus's stub provenance. Collisions get a numeric suffix, never an
overwrite; a failed export never eats its sources (archive runs strictly
after the write). `[paths] archive_dir` defaults to `Data/archive`
(`atlantis/export.py` `archive_build`, `atlantis/__main__.py`,
`tests/test_export.py`).

## 2026-07-04 ‚Äî Alias prompt: bridges, not restatements

The classifier's alias rule now explains WHY aliases exist (a reader searches
in their own words; the alias is the lexical bridge SGC's stemmed TF-IDF
retrieval needs) and forbids the useless kinds: words already in the chunk
text, inflections of them (Porter stemming matches those for free), and
generic category words (`atlantis/classify.py` `_SYSTEM_PROMPT`). Verified
against the live KoboldCPP/Gemma server on the fixture corpus: the old prompt
produced restatements like `greys: subterranean race`; the new one produces
stem-novel bridges (`steeping`/`aerating` for "let it sit"/"fluff with a
fork", `shrimping` for "hip escape", `armlock` for the kimura).

## 2026-07-04 ‚Äî Re-anchor: Atlantis is the brain-pack compiler; Chroma tail goes dormant

An audit found the codebase still describing its original mission (documents ‚Üí
Chroma vector DB) everywhere except `export.py`, and every ingest paying the
chromadb + MiniLM embedding cost for vectors that never reach a pack. Fixed in
both directions ‚Äî identity and cost.

### Changed
- **Default `ingest` is now the lean compile path**: discover ‚Üí chunk ‚Üí
  classify ‚Üí assemble ‚Üí write chunk files ‚Üí stamp `backend` into `index.json`.
  The original tail (TF-IDF salience ‚Üí related ‚Üí categorical index ‚Üí
  Chroma/MiniLM upsert) runs only under the new **`ingest --full`** flag,
  retained as dormant Phase 2b infrastructure (the future embeddings-vs-lexical
  comparison; `sgc-brain/2` salience weighting). Skipped stages report
  `done skipped (compile mode; --full restores)` so the dashboard never hangs
  on a waiting row. The `backend` stamp ‚Äî the one key `export` reads for
  `stub` provenance ‚Äî is written in BOTH modes (`atlantis/pipeline.py`).
- **Phase 2b imports are lazy**: `salience`/`index_builder` (numpy) and
  `chroma_writer` (chromadb + ONNX) import only inside `--full` branches. A
  compile-only environment needs just PyYAML (+ requests for non-stub
  enrichment).
- **`doctor` reworked around pack readiness**: chunk-file count, `index.json`
  backend stamp (with the stub value packs would export), raw-doc count. Only
  the compile-path dependency (yaml) gates the exit code; KoboldCPP
  reachability and the numpy/chromadb/requests deps are reported as
  informational (`--stub` needs no model).
- **Docs re-anchored** on the compiler mission: `README.md` (rewritten ‚Äî
  compile-loop usage, the re-ingest-destroys-hand-edits warning, dormant-tail
  framing), `CLAUDE.md` (identity, stack tiers, verification convention now
  "verify against the pack contract"), `config/atlantis.toml` comments,
  `atlantis-salience-schema-v1.md` consumers line + Chroma-mapping banner,
  module docstrings (`__init__`, `pipeline`, `chroma_writer`, `reporting`),
  CLI help (`--no-files` now warns it starves export; `query` labeled
  Phase 2b).

### Added
- `tests/test_pipeline.py`: compile-mode test (chroma/index skipped, backend
  stamped, chunks valid) alongside the `--full` test that pins the original
  end-to-end behavior.
