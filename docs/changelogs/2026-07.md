# 2026-07

## 2026-07-04 — Re-anchor: Atlantis is the brain-pack compiler; Chroma tail goes dormant

An audit found the codebase still describing its original mission (documents →
Chroma vector DB) everywhere except `export.py`, and every ingest paying the
chromadb + MiniLM embedding cost for vectors that never reach a pack. Fixed in
both directions — identity and cost.

### Changed
- **Default `ingest` is now the lean compile path**: discover → chunk →
  classify → assemble → write chunk files → stamp `backend` into `index.json`.
  The original tail (TF-IDF salience → related → categorical index →
  Chroma/MiniLM upsert) runs only under the new **`ingest --full`** flag,
  retained as dormant Phase 2b infrastructure (the future embeddings-vs-lexical
  comparison; `sgc-brain/2` salience weighting). Skipped stages report
  `done skipped (compile mode; --full restores)` so the dashboard never hangs
  on a waiting row. The `backend` stamp — the one key `export` reads for
  `stub` provenance — is written in BOTH modes (`atlantis/pipeline.py`).
- **Phase 2b imports are lazy**: `salience`/`index_builder` (numpy) and
  `chroma_writer` (chromadb + ONNX) import only inside `--full` branches. A
  compile-only environment needs just PyYAML (+ requests for non-stub
  enrichment).
- **`doctor` reworked around pack readiness**: chunk-file count, `index.json`
  backend stamp (with the stub value packs would export), raw-doc count. Only
  the compile-path dependency (yaml) gates the exit code; KoboldCPP
  reachability and the numpy/chromadb/requests deps are reported as
  informational (`--stub` needs no model).
- **Docs re-anchored** on the compiler mission: `README.md` (rewritten —
  compile-loop usage, the re-ingest-destroys-hand-edits warning, dormant-tail
  framing), `CLAUDE.md` (identity, stack tiers, verification convention now
  "verify against the pack contract"), `config/atlantis.toml` comments,
  `atlantis-salience-schema-v1.md` consumers line + Chroma-mapping banner,
  module docstrings (`__init__`, `pipeline`, `chroma_writer`, `reporting`),
  CLI help (`--no-files` now warns it starves export; `query` labeled
  Phase 2b).

### Added
- `tests/test_pipeline.py`: compile-mode test (chroma/index skipped, backend
  stamped, chunks valid) alongside the `--full` test that pins the original
  end-to-end behavior.
